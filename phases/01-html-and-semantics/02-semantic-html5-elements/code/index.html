What is Semantic HTML?

Semantic HTML uses tags that describe the meaning of content, not just appearance. Compare:

Non-Semantic Semantic Meaning
<div class="nav"> <nav> Navigation menu
<div class="article"> <article> Self-contained content
<div class="sidebar"> <aside> Tangential content
<b> <strong> Importance/urgency
<i> <em> Emphasis/stress

Screen readers announce semantic tags. Search engines understand them. Future developers (and you, six months later) can read the code and understand structure immediately.

Layout Landmarks

These structural elements define the major regions of a page:

Landmark usage:

  • <header>: Page header or section header (can use multiple)
  • <nav>: Navigation menu (use multiple for different menus)
  • <main>: Primary content (use only once per page)
  • <section>: Thematic grouping of content
  • <article>: Self-contained, reusable content (blog post, news item)
  • <aside>: Tangential content (sidebar, related links)
  • <footer>: Page footer or section footer (can use multiple)

Figures and Captions

Use <figure> and <figcaption> to associate images, diagrams, or code examples with their captions.

Text-Level Semantics

Distinguish between physical and logical formatting:

Tag Use Case Example
<strong> Importance/urgency Warning: This will delete your data.
<em> Emphasis/stress I really love this feature.
<code> Computer code const x = 42;
<kbd> User input (keyboard) Press Ctrl + C
<samp> Program output $ npm run build
<mark> Highlighted/relevant Find the important part.
<cite> Citation/reference According to Web Standards Today
<time> Date/time May 13, 2024
