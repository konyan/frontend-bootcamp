Embed, optimize, and make your media accessible

Responsive Images

Serve different image sizes to different devices. The browser downloads only what's needed, improving performance.

Example: Responsive Image with srcset

Key attributes:
  • src: Fallback image for old browsers
  • srcset: List of images and their widths
  • sizes: Media queries telling browser what size the image will be
  • alt: Description for screen readers and broken images

Art Direction with <picture>

Use <picture> when you need different crops or compositions for different viewports (e.g., portrait on mobile, landscape on desktop). 

<picture>
  <source media="(max-width: 600px)" 
          srcset="portrait-small.jpg 320w, portrait-large.jpg 600w">
  <source media="(min-width: 601px)" 
          srcset="landscape.jpg 1200w">
  <img src="landscape.jpg" alt="Team photo">
</picture>

Modern Image Formats

Use <picture> with type attribute to load modern formats (WebP, AVIF) on supported browsers, with JPEG fallback:

<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Description">
</picture>

Video Embedding

Use the native <video> element for full control over playback, captions, and accessibility.

Basic HTML5 Video

<video controls width="640" height="360" poster="thumbnail.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  <p>Your browser doesn't support HTML5 video.
     <a href="video.mp4">Download instead</a>.</p>
</video>

Video with Captions

Always provide captions for accessibility. Use WebVTT (.vtt) files linked with <track> elements. 

<video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="captions" src="captions.en.vtt" srclang="en" label="English">
  <track kind="captions" src="captions.es.vtt" srclang="es" label="Español">
  <p>Your browser doesn't support this video.</p>
</video>
Key attributes:
  • controls: Show play/pause, volume buttons
  • poster: Image shown before playback starts
  • width/height: Dimensions (aspect ratio reserve space)
  • <source>: Multiple formats for browser compatibility
  • <track>: Caption files (VTT format)

Track Kinds

Kind Purpose
captions Dialogue + sound effects (for deaf users)
subtitles Dialogue translation (no sound effects)
descriptions Audio descriptions for blind users
chapters Chapter markers for navigation

Audio Embedding

Use <audio> for podcasts, background music, or voice content. Always provide a transcript for accessibility.

Basic Audio

<audio controls>
  <source src="podcast.mp3" type="audio/mpeg">
  <source src="podcast.ogg" type="audio/ogg">
  <p>Your browser doesn't support this audio.
     <a href="podcast.mp3">Download instead</a>.</p>
</audio>

Audio with Transcript

Progressive Disclosure with <details>

The <details> and <summary> elements create expandable sections without JavaScript. Screen readers understand them natively.

When to use Details Elements
  • FAQs (each Q&A in its own details block)
  • Extended transcripts or descriptions
  • Course prerequisites or requirements
  • Additional information not needed on page load
  • Progressive disclosure patterns
Browser Support

<details> is supported in all modern browsers. For Internet Explorer, use JavaScript to provide fallback functionality.

CSS Styling Tips
  • Use summary { cursor: pointer; } to show clickability
  • Style details[open] differently when expanded
  • Use ::marker pseudo-element to customize the triangle
  • Add transitions for smooth expand/collapse animation

Multimedia Accessibility Checklist

Images

  • ☑️ Every image has meaningful alt text
  • ☑️ Decorative images use empty alt=""
  • ☑️ Complex images have longer descriptions
  • ☑️ Using srcset for responsive sizing

Video

  • ☑️ Video has captions for dialogue and sound effects
  • ☑️ Captions use proper timing and punctuation
  • ☑️ Audio descriptions available for visual content
  • ☑️ poster image shows accessible preview

Audio

  • ☑️ Podcast/interview has full transcript
  • ☑️ Transcript is searchable and linked from page
  • ☑️ Speaker names clearly marked in transcript
  • ☑️ Timestamps included for longer content

Interactive Media

  • ☑️ Keyboard accessible (Tab, Enter, Escape)
  • ☑️ Focus indicators visible
  • ☑️ ARIA labels for custom controls
  • ☑️ Tested with screen reader
