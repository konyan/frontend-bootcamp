# OS, Shell & Workflow Automation

> If every command feels longer than it should, your workflow stays slow no matter how good the framework is.

**Type:** Build
**Languages:** Shell
**Prerequisites:** None
**Time:** ~60 minutes

## Learning Objectives

- Customize your shell startup files for daily frontend work
- Manage environment variables, PATH, and command aliases safely
- Automate repetitive tasks with small shell functions
- Understand dotfiles as part of project-level craftsmanship

## The Concept

The shell is your command center. Keep it small, repeatable, and tuned to the commands you use most.

## Build It

### Step 1
Add a few aliases and shell functions to your profile.

### Step 2
Export a project-specific environment variable.

### Step 3
Verify that your PATH resolves the tools you expect.

### Step 4
Store the settings in a dotfiles repo or starter template.

## Use It

Apply the workflow to your current frontend project instead of waiting until later.

## Exercises

1. Create one alias and one shell function for frontend work.
1. Add a temporary environment variable and print it back out.
1. Check which version of node and pnpm your shell resolves first.
