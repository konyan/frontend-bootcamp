# Advanced Git Workflow

> Basic push and pull is not enough once a project has multiple contributors and active branches.

**Type:** Learn
**Languages:** —
**Prerequisites:** None
**Time:** ~75 minutes

## Learning Objectives

- Use rebase, cherry-pick, and stash confidently
- Resolve conflicts without losing work
- Inspect history with log, reflog, and diff tools
- Understand when to rewrite history and when not to

## The Concept

Git is a history tool as much as a file sync tool. Learn to move carefully and inspect before you change.

## Build It

### Step 1
Rebase a small feature branch onto main.

### Step 2
Cherry-pick one commit into another branch.

### Step 3
Resolve a merge conflict and review the diff afterward.

### Step 4
Use reflog to recover from a mistaken reset or checkout.

## Use It

Apply the workflow to your current frontend project instead of waiting until later.

## Exercises

1. Create a practice repo and rebase two commits.
1. Cherry-pick a small change into a new branch.
1. Describe when you would avoid rewriting shared history.
