# Deployment & Preview Workflow

> A project is not really ready until you can build it, preview it, and share the result with confidence.

**Type:** Build
**Languages:** TypeScript
**Prerequisites:** None
**Time:** ~60 minutes

## Learning Objectives

- Understand preview deployments and production promotion
- Work with environment variables for deployed apps
- Use a build-and-preview loop before merging changes
- Recognize how frontend hosting platforms fit into the workflow

## The Concept

A deployment workflow should make it easy to see the app in a realistic environment before it reaches users.

## Build It

### Step 1
Run a production build locally and inspect the output.

### Step 2
Deploy a preview to Vercel or a similar host.

### Step 3
Check that env vars and base URLs are configured correctly.

### Step 4
Share the preview link and verify the deployment feedback loop.

## Use It

Apply the workflow to your current frontend project instead of waiting until later.

## Exercises

1. Compare local preview and production deployment behavior.
1. Write down the env vars needed for one sample frontend app.
1. Explain why preview URLs help code review.
