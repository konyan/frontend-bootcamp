# Essential Extensions & Workspace Hygiene

> Too many extensions and too many local overrides create inconsistent behavior that is hard to debug later.

**Type:** Build
**Languages:** TypeScript
**Prerequisites:** None
**Time:** ~45 minutes

## Learning Objectives

- Pick the extensions that matter for frontend productivity
- Avoid extension bloat and keep the workspace lean
- Use Prettier, ESLint, and Tailwind IntelliSense intentionally
- Document shared project settings in the repository

## The Concept

The best workspace hygiene keeps the important tools obvious and the rest out of the way.

## Build It

### Step 1
Install the project extensions and confirm they activate where expected.

### Step 2
Create a .vscode/extensions.json recommendation list.

### Step 3
Remove anything that duplicates formatter or linter behavior.

### Step 4
Use settings that match the project rather than personal habits.

## Use It

Apply the workflow to your current frontend project instead of waiting until later.

## Exercises

1. Write a short extension recommendation list for a frontend starter.
1. Compare your personal settings with a workspace settings file.
1. Turn off one extension that conflicts with Prettier or ESLint.
