# TypeScript Configuration & Path Aliases

> Without a clear TypeScript configuration, a project can become permissive in the wrong places and awkward to navigate.

**Type:** Build
**Languages:** TypeScript
**Prerequisites:** None
**Time:** ~75 minutes

## Learning Objectives

- Create a strict tsconfig for safer frontend code
- Use path aliases like @/* to simplify imports
- Understand project references at a practical level
- Recognize the most useful strictness flags for teams

## The Concept

TypeScript works best when it is opinionated enough to catch mistakes but still easy to read and maintain.

## Build It

### Step 1
Enable a strict configuration and inspect the compiler options.

### Step 2
Add a path alias and test it in one import statement.

### Step 3
Review why noImplicitAny and strictNullChecks matter.

### Step 4
Map how the editor and the compiler use the same config.

## Use It

Apply the workflow to your current frontend project instead of waiting until later.

## Exercises

1. Add a path alias to a small TypeScript project.
1. Turn on strict mode and fix the first few errors.
1. Explain why a shared tsconfig helps a team stay consistent.
