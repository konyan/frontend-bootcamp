# The Modern Development Environment

> A clean frontend setup removes the friction of missing tools, inconsistent editor behavior, and mismatched runtime versions.

**Type:** Build
**Languages:** Shell
**Prerequisites:** None
**Time:** ~60 minutes

## Learning Objectives

- Set up a frontend workspace that stays fast and predictable
- Configure shell aliases, editor defaults, and essential extensions
- Verify Node.js, pnpm, and git from the terminal
- Understand the layers that make a project environment repeatable

## The Concept

Treat the environment as part of the project. The shell, editor, runtime, and project tools should all reinforce the same workflow.

## Build It

### Step 1
Create a few shell aliases for commands you run every day.

### Step 2
Add workspace settings for formatting and type tooling in VS Code.

### Step 3
Install the recommended extensions and confirm they are enabled.

### Step 4
Run the verification script to check the runtime stack.

## Use It

Apply the workflow to your current frontend project instead of waiting until later.

## Exercises

1. Add three shell aliases you will actually use.
1. Create a .vscode/settings.json file for a frontend project.
1. Install the recommended extensions and verify they are active.
